## Zenn 記事（エンジニア向け）

### タイトル案
「自分の結婚式のために AI 笑顔採点サービスを個人開発した話」

### 構成

#### 1. はじめに（動機）
- 自分の結婚式で「全員が参加できる余興」がほしかった
- エムスリーテックブック8の記事に触発された
- 元のAWS実装をGCPで再構築した経緯

#### 2. サービス概要
- ゲストがLINEで写真を送る → AIが笑顔を採点 → リアルタイムランキング
- デモ動画やスクショを挿入
- スコアリングの仕組み（Vision API + Gemini + Average Hash）

#### 3. 技術スタック
- LINE Messaging API（Webhook → Cloud Functions）
- Cloud Vision API（顔検出・笑顔度）
- Vertex AI Gemini 2.5 Flash（テーマ評価）
- Firestore（リアルタイムDB）
- Cloud Storage（画像保存、署名付きURL）
- Firebase Hosting（フロントエンド）
- Terraform（IaC）

#### 4. 設計で悩んだポイント
- **スコアリングアルゴリズム**: グループ写真を促す設計（人数×笑顔度の合計）
- **類似画像ペナルティ**: Average Hash + Hamming距離で連写を抑制
- **Geminiの選定理由**: GPT-4oだとスコアが甘くなる問題
- **マルチテナント設計**: 1つのLINE Botで複数イベントを扱う
- **フォールバック設計**: API障害時のグレースフルデグラデーション

#### 5. 実際の結婚式で使ってみた結果
- 80名規模、368枚、参加率60%
- ゲストの反応
- 想定外だったこと・改善点

#### 6. コスト
- 1イベントあたり$10-30
- 無料枠を最大限活用

#### 7. まとめ・今後
- 一般向けサービスとして公開中
- サービスURL

### タグ
`個人開発` `GCP` `LINE Bot` `AI` `結婚式` `Firebase` `Terraform`

## note 記事（一般向け）

### note タイトル案
「エンジニアの僕が自分の結婚式のために作った『AI笑顔写真コンテスト』の話」

### note 構成

#### 1. きっかけ
- 結婚式の余興、何しよう問題
- 「全員が自然に参加できる」余興を作りたかった
- 写真共有サービスはあるけど「ゲーム性」がない

#### 2. どんなサービス？
- LINEで写真を送る → AIが笑顔を採点 → ランキング表示
- 技術的な話は最小限に、体験ベースで説明
- スクショや写真で視覚的に伝える

#### 3. 実際の結婚式での様子
- 準備（QRコードを印刷して配布）
- 当日の流れ
- ゲストの反応（盛り上がったシーン）
- 数字で振り返り（368枚、60%参加率）

#### 4. 作って良かったこと
- 写真が集まった（普通なら届かない写真がたくさん）
- ゲスト同士が交流していた
- 「結婚式の思い出」がデータとして残った

#### 5. サービスとして公開しました
- 他のカップルにも使ってほしい
- 現在は無料で提供中
- サービスURL + 申し込み方法

### ハッシュタグ / カテゴリ
`結婚式` `余興` `個人開発` `AI` `ウェディング`
