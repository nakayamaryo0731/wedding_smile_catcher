# プロダクト公開ロードマップ

Wedding Smile Catcher を一般向けサービスとして公開するための全体方針をまとめたドキュメント。

最終更新: 2026-01-26

---

## 1. ビジネス方針

### ターゲット顧客

- **B2C**: 結婚式を控えるカップル / 夫婦
- 一旦は結婚式特化（機能的には他パーティーにも横展開可能）

### 価格設定

| 項目 | 内容 |
|------|------|
| 価格 | **15,000円/イベント（税込）** |
| 課金タイミング | イベント作成時 |
| 含まれるもの | 全機能利用、画像保存（イベント後30日間） |

### コスト構造

目標: 顧客がいない月は固定費0円

| 状況 | 固定費 | 変動費 | 計 |
|------|--------|--------|-----|
| 顧客0の月 | 0円 | 0円 | **0円** |
| 顧客1件の月 | 5,000円（LINE） | 1,000円 | **6,000円** |
| 顧客3件の月 | 5,000円 | 3,000円 | **8,000円** |

- LINE公式アカウント: イベントがある月のみライトプラン（5,000円/月）にアップグレード
- ドメイン: 初期はFirebaseデフォルトドメイン（`xxx.web.app`）を使用
- インフラ: GCP無料枠を最大限活用

### 収益シミュレーション

| 月間イベント数 | 売上 | コスト | 粗利 |
|---------------|------|--------|------|
| 1件 | 15,000円 | 6,000円 | **9,000円** |
| 3件 | 45,000円 | 8,000円 | **37,000円** |
| 5件 | 75,000円 | 10,000円 | **65,000円** |

## 2. 販売チャネル

### Phase 1: 実績作り（0〜3件）

- 知人・友人の結婚式で使ってもらう（無料 or 割引）
- 事例・スクリーンショットを集める
- 使用感のフィードバックを収集

### Phase 2: 小規模販売（3〜10件）

- **ココナラ**に出品（集客をプラットフォームに任せる）
- 手数料22%（15,000円 → 手取り約11,700円）は「広告費」と割り切る
- 評価・レビューを貯めて信頼性を構築

### Phase 3: 自前チャネル（10件〜）

- LP作成
- ココナラの実績・レビューを引用
- SNS（Instagram等）での発信を開始
- ココナラと自前の併用

## 3. サポート・リスク対応方針

### サポート体制

| 項目 | 方針 |
|------|------|
| 当日リアルタイムサポート | **提供しない**（利用規約に明記） |
| 事前テスト | **必須**（イベント1週間前までに完了すること） |
| 問い合わせ対応 | メールベース、平日対応 |

### 返金ポリシー

| ケース | 対応 |
|--------|------|
| 明らかなシステム障害（当社起因） | **全額返金** |
| GCP / LINE API の障害（外部起因） | **要検討**（半額返金 or 免責）。フォールバック設計により完全停止の可能性は低い（`scoring.md` 参照） |
| 顧客の設定ミス・操作ミス | **返金なし**（事前テストで防げたはず） |
| イベント7日前までのキャンセル | **全額返金** |
| イベント7〜3日前のキャンセル | **半額返金** |
| イベント3日前以降のキャンセル | **返金なし** |

### フォールバック設計

- **スコアリングのグレースフルデグラデーション**: 外部API（Vision API / Gemini）が障害時でも、生存している方の実測値 + 乱数で採点を継続。詳細は `docs/architecture/scoring.md` を参照
- 万一システム全体が動かない場合、LINEグループでの写真共有を推奨
- 利用ガイドに「緊急時の対処法」を記載

## 4. バイラル・成長戦略

### 採用する施策

| 施策 | タイミング | 内容 |
|------|-----------|------|
| ランキング画面にサービスロゴ | イベント中 | 式場スクリーンで自然に認知（控えめに表示） |

- スコアカード画像生成
- 紹介割引コード（Phase 3以降で再検討）

## 5. 技術方針

### LINE Bot 戦略

共通Bot + ディープリンク方式を採用。

- 1つのLINE公式アカウントで全イベントを管理
- QRコードにイベントコードを埋め込んだディープリンクを使用
- ゲスト体験: QRスキャン → 送信ボタン → 名前入力 → 写真投稿

```
ディープリンク例:
https://line.me/R/oaMessage/@xxx/?text=JOIN%20{event_code}
```

### マルチテナント対応

- 現状: `CURRENT_EVENT_ID` 環境変数で1イベントのみ
- 目標: 複数顧客の同時イベント開催を可能に
- 詳細設計: `docs/planning/multi-tenant-design.md` を参照

### 認証基盤

- 現状: 管理画面はハードコードパスワード（SHA-256）
- 目標: 顧客ごとのアカウント管理
- 候補: Firebase Authentication
