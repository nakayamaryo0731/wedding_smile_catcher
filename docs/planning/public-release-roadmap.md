# 一般向けリリースに向けたロードマップ

このドキュメントは、Wedding Smile Catcher を一般向けサービスとしてリリースする場合に必要な要素をまとめたものです。

## 現状サマリー

| 項目 | 実装度 | 状態 |
|------|--------|------|
| コア機能（スコアリング、LINE Bot、ランキング表示） | 95% | ほぼ完成 |
| マルチテナント | 60% | 部分対応（並行イベント不可） |
| 認証・認可 | 70% | 基本的（管理画面セキュリティ弱い） |
| 管理画面 | 85% | 充実 |
| 課金・支払い | 0% | 未実装 |
| ドキュメント | 80% | 概ね良好 |
| テスト | 60% | Unit/Integration あり、E2E なし |
| インフラ自動化 | 90% | Terraform 完成、CD 部分的 |

---

## 必要な要素

### 🔴 必須（これがないとサービスとして成立しない）

#### 1. マルチテナント完全対応

- **現状**: `CURRENT_EVENT_ID` 環境変数で1イベントのみ
- **必要**: 複数の顧客が同時にイベント開催可能に
- **実装案**:
  - LINE Bot を顧客ごとに発行
  - または リッチメニューでイベント選択

#### 2. ユーザー認証基盤

- **現状**: 管理画面はハードコードパスワード
- **必要**: 顧客ごとのアカウント管理
- **候補**: Firebase Auth / Auth0 / Supabase Auth

#### 3. 課金・支払いシステム

- **現状**: 未実装
- **必要**: 顧客からの料金徴収
- **候補**: Stripe（イベント単位 or サブスク）
- **想定価格帯**: 1イベント 3,000〜10,000円？

#### 4. オンボーディングUI

- **現状**: CLI スクリプトでイベント作成
- **必要**: 顧客がセルフサービスでセットアップ
  - アカウント作成
  - イベント作成（日時、テーマ設定）
  - LINE Bot 接続 or QRコード発行
  - ランキング画面URL発行

#### 5. 利用規約・プライバシーポリシー

- 顔画像を扱うため特に重要
- GDPR / 個人情報保護法対応

---

### 🟡 重要（サービス品質に直結）

#### 6. 顧客向けダッシュボード

- イベント管理（作成・編集・アーカイブ）
- リアルタイム統計
- 画像一括ダウンロード
- 参加者管理

#### 7. E2Eテスト・監視

- **現状**: Unit/Integration のみ
- **必要**: 本番環境の監視、アラート設定
- **候補**: Cloud Monitoring / Sentry など

#### 8. ステージング環境

- **現状**: 本番のみ
- **必要**: 顧客向けデモ環境、新機能テスト環境

#### 9. カスタマイズ機能

- テーマカラー変更
- ランキング表示のカスタマイズ
- AIの評価観点の変更

---

### 🟢 あると良い（差別化要素）

#### 10. LINE公式アカウント共有 or 自動発行

- 顧客が自分のLINE Botを用意しなくて良い仕組み
- LINE Messaging API の Business Connect 検討

#### 11. 多言語対応

- AIコメントの言語切り替え
- UIの国際化（i18n）

#### 12. イベント後レポート

- 自動生成される思い出レポート
- 統計サマリー PDF エクスポート

#### 13. 他サービス連携

- 写真共有サービス連携
- SNS投稿機能

## ビジネスモデル候補

| モデル | 概要 | メリット | デメリット |
|--------|------|----------|------------|
| イベント単価 | 1イベント5,000円など | シンプル、理解しやすい | 顧客獲得コスト高 |
| サブスク | 月額制（式場向け） | 安定収益 | 個人利用には不向き |
| フリーミアム | 基本無料＋上位機能有料 | 利用ハードル低 | 収益化まで時間 |
