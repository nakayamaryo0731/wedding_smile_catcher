# テストデータ

Wedding Smile Catcher プロジェクトのテスト用画像データ

---

## 必要な画像

以下の画像を準備してください：

### 1. `happy_couple.jpg`

- **用途**: 基本的なテスト画像、E2Eテスト
- **内容**: 2人の笑顔（カップル）
- **期待されるスコア**: 150-200点

### 2. `happy_group_5.jpg`

- **用途**: 高スコアテスト
- **内容**: 5人の笑顔グループ
- **期待されるスコア**: 400-500点

### 3. `single_smile.jpg`

- **用途**: 低スコアテスト
- **内容**: 1人の笑顔
- **期待されるスコア**: 70-95点

### 4. `no_smile.jpg`

- **用途**: 笑顔なしテスト
- **内容**: 笑顔のない人物
- **期待されるスコア**: 0-50点

### 5. `off_theme.jpg`

- **用途**: AI評価（テーマ外）テスト
- **内容**: 結婚式と関係ない画像（風景など）
- **期待されるAI評価**: 20-40点

---

## 画像の準備方法

### オプション1: 実際の写真を使用

- スマホで撮影した写真をこのディレクトリにコピー
- ファイル名を上記に合わせてリネーム

### オプション2: サンプル画像をダウンロード

- [Unsplash](https://unsplash.com/) などから無料の写真をダウンロード
- キーワード例:
  - "happy couple" → `happy_couple.jpg`
  - "group of friends smiling" → `happy_group_5.jpg`
  - "portrait smile" → `single_smile.jpg`

---

## 画像のガイドライン

- **フォーマット**: JPG または PNG
- **サイズ**: 最大 10MB（LINE Botの制限）
- **解像度**: 最低 800×600ピクセル推奨
- **内容**: 顔が明確に写っていること

---

## 注意事項

- テスト画像は `.gitignore` に追加されています（プライバシー保護のため）
- 実際の人物が写っている場合は、本人の許可を得てください
- 本番環境でテストする際は、実際のゲストに協力を依頼してください

---

## 現在の状態

```text
tests/test_data/
├── README.md (このファイル)
└── (画像をここに配置してください)
```

画像が準備できたら、以下のコマンドでテストを実行できます：

```bash
# ユニットテスト
pytest tests/unit/ -v

# 統合テスト
pytest tests/integration/ -v

# E2Eテスト（半自動）
pytest tests/e2e/ -v
```
